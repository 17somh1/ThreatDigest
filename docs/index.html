The New Phishing Click: How OAuth Consent Bypasses MFA

The emergence of the EvilTokens phishing platform poses a significant threat to organizations using Microsoft 365, potentially bypassing multi-factor authentication (MFA) and leading to unauthorized access.

Who should care: IT security teams and administrators managing Microsoft 365 environments are most exposed to this threat.
Attack path: Attackers leverage OAuth consent screens to deceive users into granting access to their accounts, effectively bypassing traditional MFA protections.
Why this matters now: Useful context for teams tracking where attacker pressure is building.
In plain English: A new phishing service is exploiting OAuth to bypass MFA for Microsoft 365 accounts, affecting hundreds of organizations.
phishing oauth mfa
  • In February 2026, a phishing-as-a-service platform named EvilTokens was launched, targeting Microsoft 365 users. Within five weeks, it successfully compromised over 340 organizations across five countries by tricking users into entering codes that appeared to be part of legitimate MFA processes.
  • This attack method undermines the security of MFA, which is a critical defense against unauthorized access. Organizations relying on Microsoft 365 must be vigilant, as compromised accounts can lead to data breaches and operational disruptions.
  • Attackers leverage OAuth consent screens to deceive users into granting access to their accounts, effectively bypassing traditional MFA protections.
  • Watch next: Monitor for updates from Microsoft regarding security patches or recommendations related to OAuth vulnerabilities.
Key terms
  • OAuth - an open standard for access delegation commonly used for token-based authentication.
  • MFA (Multi-Factor Authentication) - a security measure that requires two or more verification methods to gain access to an account.
Defender focus
  • Implement monitoring for unusual login attempts from unfamiliar locations.
  • Review and restrict OAuth app permissions within Microsoft 365 environments.
  • Educate users on recognizing phishing attempts that mimic legitimate MFA processes.
Do now
  • Enhance user training on identifying phishing attempts.
  • Audit OAuth applications currently authorized in your Microsoft 365 environment.
  • Stay updated on threat intelligence regarding phishing tactics targeting MFA.
