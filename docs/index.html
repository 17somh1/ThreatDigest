Over 400 Arch Linux packages compromised to push rootkit, infostealer

Who should care: Development and IT teams using Arch Linux, especially those managing user credentials and sensitive data.
Attack path: Attackers leverage compromised packages to install malware on user systems, allowing them to steal credentials and access tokens. This can lead to further exploitation of user accounts.
Why this matters now: Multiple sources point to the same operational problem.
