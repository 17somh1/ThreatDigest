Ghost CMS CVE-2026-26980 Exploited to Hijack 700+ Sites for ClickFix Attacks

Over 700 websites are compromised due to a critical vulnerability in Ghost CMS, risking user data and site integrity.

Who should care: Web development and security teams managing Ghost CMS installations are most exposed.
Attack path: Attackers gain leverage by exploiting the SQL injection vulnerability to execute arbitrary code, allowing them to manipulate site content and redirect users.
Why this matters now: Useful context for teams tracking where attacker pressure is building.
In plain English: A serious flaw in Ghost CMS is being actively exploited, allowing attackers to hijack sites for malicious purposes.
cve ghost cms web security
  • Threat actors are exploiting CVE-2026-26980, a critical SQL injection vulnerability in Ghost CMS's Content API. This flaw allows unauthenticated attackers to inject malicious JavaScript code into over 700 websites, facilitating ClickFix attacks.
  • The exploitation of this vulnerability can lead to significant data breaches and loss of user trust, as compromised sites may serve malicious content. Organizations using Ghost CMS need to act quickly to mitigate risks associated with this vulnerability.
  • Attackers gain leverage by exploiting the SQL injection vulnerability to execute arbitrary code, allowing them to manipulate site content and redirect users.
  • Watch next: Monitor for updates from Ghost CMS regarding patches and any further exploitation reports.
Key terms
  • SQL Injection - a type of attack that allows an attacker to interfere with the queries that an application makes to its database.
  • ClickFix - a type of attack where malicious scripts are injected into websites to manipulate user interactions.
Defender focus
  • Implement web application firewalls to detect and block SQL injection attempts.
  • Regularly update Ghost CMS to the latest version to ensure vulnerabilities are patched.
  • Conduct a security audit of all websites running Ghost CMS to identify and remediate any exploitation.
Do now
  • Patch Ghost CMS immediately to close the vulnerability.
  • Monitor web traffic for unusual activity indicative of exploitation.
  • Educate development teams on secure coding practices to prevent similar vulnerabilities.
Watch next: Monitor for updates from Ghost CMS regarding patches and any further exploitation reports.

FBI warns of Kali365 phishing service targeting Microsoft 365 accounts

The rise of the Kali365 phishing service poses an immediate threat to Microsoft 365 accounts, potentially compromising sensitive organizational data.

Who should care: IT security teams managing Microsoft 365 environments should prioritize this threat.
Attack path: Attackers can gain access by tricking users into authenticating through the phishing service, allowing them to capture session tokens and bypass MFA protections.
Why this matters now: Useful context for teams tracking where attacker pressure is building.
In plain English: A new phishing service is targeting Microsoft 365 accounts, exploiting OAuth to bypass security measures.
phishing microsoft365 oauth
  • The FBI has issued a warning about the Kali365 phishing-as-a-service platform that targets Microsoft 365 accounts. This service leverages OAuth device code authentication to steal session tokens, allowing attackers to bypass multi-factor authentication.
  • Organizations using Microsoft 365 are at significant risk of account hijacking, which could lead to data breaches and unauthorized access to sensitive information. The ability to bypass MFA undermines a critical layer of security, making it easier for attackers to compromise accounts.
  • Attackers can gain access by tricking users into authenticating through the phishing service, allowing them to capture session tokens and bypass MFA protections.
  • Watch next: Monitor for any updates from Microsoft regarding countermeasures or patches related to this phishing service.
Key terms
  • Phishing - a method of attempting to acquire sensitive information by masquerading as a trustworthy entity in electronic communications.
  • OAuth - an open standard for access delegation commonly used as a way to grant websites or applications limited access to user information without exposing passwords.
Defender focus
  • Implement monitoring for unusual login attempts to Microsoft 365 accounts.
  • Review and enhance MFA configurations to ensure they are robust against bypass techniques.
  • Educate users on recognizing phishing attempts, especially those that exploit OAuth.
Do now
  • Increase monitoring for suspicious activity in Microsoft 365 accounts.
  • Review and strengthen MFA policies to mitigate bypass risks.
  • Conduct a security awareness session focused on phishing tactics.
Watch next: Monitor for any updates from Microsoft regarding countermeasures or patches related to this phishing service.
