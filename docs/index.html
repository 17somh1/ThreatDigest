Russian CTRL Toolkit Delivered via Malicious LNK Files Hijacks RDP via FRP Tunnels

The emergence of the Russian CTRL toolkit poses a significant threat to organizations by enabling RDP hijacking and credential theft through seemingly innocuous files.

Who should care: IT security teams and system administrators managing remote access solutions should be particularly vigilant.
Attack path: Attackers can leverage malicious LNK files to execute the CTRL toolkit on victim machines, enabling them to hijack RDP sessions and steal credentials.
Why this matters now: Useful context for teams tracking where attacker pressure is building.
In plain English: A new Russian toolkit is being spread via malicious shortcut files, allowing attackers to hijack remote desktop connections and steal credentials.
ransomware cve cloud
  • Researchers identified a remote access toolkit known as CTRL, which is distributed through malicious Windows shortcut (LNK) files disguised as private key folders. This toolkit is built using .NET and includes functionalities for credential phishing, keylogging, RDP hijacking, and reverse tunneling.
  • The ability to hijack RDP sessions can lead to unauthorized access to critical systems, potentially resulting in data breaches or system compromises. Organizations that rely on RDP for remote access are particularly vulnerable, as attackers can exploit this toolkit to gain control over their environments.
  • Attackers can leverage malicious LNK files to execute the CTRL toolkit on victim machines, enabling them to hijack RDP sessions and steal credentials.
  • Watch next: Monitor for any unusual RDP access patterns and updates from security vendors regarding this toolkit.
Key terms
  • RDP - Remote Desktop Protocol, a protocol used for remotely accessing computers.
  • LNK - A file extension for Windows shortcut files that can point to other files or programs.
Defender focus
  • Implement monitoring for LNK file execution in user directories.
  • Review RDP access logs for any unauthorized login attempts.
  • Educate users on the risks of opening unknown shortcut files.
Do now
  • Enhance email filtering to block malicious LNK files.
  • Conduct a security audit of RDP configurations and access controls.
  • Deploy endpoint protection solutions that can detect and block the CTRL toolkit.
Watch next: Monitor for any unusual RDP access patterns and updates from security vendors regarding this toolkit.
