29-Year-Old Squid Proxy Bug 'Squidbleed' Can Leak Cleartext HTTP Requests

The Squidbleed vulnerability exposes sensitive user data, including credentials, to unauthorized users of the same proxy, posing a significant risk to data confidentiality.

Who should care: Network and security teams managing systems that utilize the Squid web proxy should prioritize this issue.
Attack path: An attacker can exploit this vulnerability by sending traffic through the same Squid proxy, allowing them to intercept and read cleartext HTTP requests from other users.
Why this matters now: Useful context for teams tracking where attacker pressure is building.
In plain English: A long-standing vulnerability in the Squid web proxy can leak sensitive HTTP requests to other users sharing the same proxy.
cve proxy data leakage
  • A vulnerability known as Squidbleed was disclosed by researchers at Calif.io, stemming from a heap over-read issue in the Squid web proxy. This flaw allows one user to access another user's cleartext HTTP requests, which may include sensitive information such as credentials and session tokens.
  • This vulnerability can lead to unauthorized access to user accounts and sensitive data, especially in environments where multiple users share the same proxy. Given the age of the bug, many systems may still be using vulnerable configurations, increasing the risk of exploitation.
  • An attacker can exploit this vulnerability by sending traffic through the same Squid proxy, allowing them to intercept and read cleartext HTTP requests from other users.
  • Watch next: Monitor for updates from Squid maintainers regarding patches or configuration changes to mitigate this vulnerability.
Key terms
  • HEAP OVER-READ - a type of memory vulnerability where an application reads more data from memory than it should, potentially exposing sensitive information.
  • CLEARTEXT - data that is transmitted without encryption, making it readable by anyone who intercepts it.
Defender focus
  • Review Squid proxy configurations to ensure they are not using default settings that expose this vulnerability.
  • Implement network segmentation to limit access to shared proxies.
  • Monitor proxy logs for unusual access patterns that may indicate exploitation attempts.
Do now
  • Patch the Squid proxy to the latest version as soon as possible.
  • Reconfigure proxy settings to enhance security and limit exposure.
  • Educate users about the risks of using shared proxies for sensitive transactions.
Watch next: Monitor for updates from Squid maintainers regarding patches or configuration changes to mitigate this vulnerability.
