Payouts King ransomware uses QEMU VMs to bypass endpoint security

The Payouts King ransomware's innovative use of QEMU VMs poses a significant threat to endpoint security, making detection and response more challenging.

Who should care: IT security teams and organizations with endpoint security solutions should be particularly vigilant.
Attack path: Attackers gain leverage by deploying QEMU VMs that operate outside the visibility of standard security protocols, allowing them to execute malicious activities without detection.
Why this matters now: Useful context for teams tracking where attacker pressure is building.
In plain English: A new ransomware variant is using virtual machines to evade security measures.
ransomware qemu endpoint security
  • Payouts King ransomware has been identified using the QEMU emulator to create hidden virtual machines on compromised systems. This technique allows it to establish a reverse SSH backdoor, effectively bypassing traditional endpoint security measures.
  • This method of operation complicates detection efforts, as the ransomware can operate within a virtual environment that may not be monitored by standard security tools. Organizations relying on conventional endpoint defenses are at increased risk of undetected ransomware activity.
  • Attackers gain leverage by deploying QEMU VMs that operate outside the visibility of standard security protocols, allowing them to execute malicious activities without detection.
  • Watch next: Monitor for updates from security vendors regarding enhanced detection capabilities for virtualized environments.
Key terms
  • QEMU - an open-source emulator that allows users to run virtual machines.
  • Reverse SSH - a method where a compromised system initiates a connection back to an attacker's server, allowing remote access.
Defender focus
  • Implement monitoring for unusual processes that may indicate the presence of virtual machines.
  • Enhance endpoint detection and response (EDR) capabilities to identify hidden processes.
  • Review and update incident response plans to account for virtualized threats.
Do now
  • Investigate any alerts related to QEMU or unusual virtual machine activity.
  • Update endpoint security solutions to ensure they can detect and respond to virtualized threats.
  • Conduct a security audit to identify potential vulnerabilities in current defenses.
Watch next: Monitor for updates from security vendors regarding enhanced detection capabilities for virtualized environments.

Mirai Variant Nexcorium Exploits CVE-2024-3721 to Hijack TBK DVRs for DDoS Botnet

The exploitation of CVE-2024-3721 poses a risk of widespread DDoS attacks, impacting network stability for affected users.

Who should care: Network security teams managing TBK DVRs and TP-Link routers should prioritize this issue.
Attack path: Attackers gain leverage by exploiting the command injection vulnerability to gain control over the devices, turning them into part of a botnet for DDoS attacks.
Why this matters now: Useful context for teams tracking where attacker pressure is building.
In plain English: A new variant of the Mirai botnet is exploiting vulnerabilities in TBK DVRs and TP-Link routers to create a DDoS threat.
ransomware cve cloud
  • Threat actors are leveraging a command injection vulnerability (CVE-2024-3721) in TBK DVRs and outdated TP-Link routers to deploy a variant of the Mirai botnet. This vulnerability has a CVSS score of 6.3, indicating a medium level of severity. The findings were reported by Fortinet FortiGuard Labs and Palo Alto Networks Unit 42.
  • If left unaddressed, these compromised devices can be used to launch large-scale DDoS attacks, affecting not only the devices themselves but also the broader network infrastructure. Organizations relying on these devices may experience service disruptions and increased operational costs due to mitigation efforts.
  • Attackers gain leverage by exploiting the command injection vulnerability to gain control over the devices, turning them into part of a botnet for DDoS attacks.
  • Watch next: Monitor for updates from vendors regarding patches for CVE-2024-3721 and any emerging threat intelligence on the Mirai variant.
Key terms
  • CVE-2024-3721 - a specific identifier for a security vulnerability that allows attackers to execute commands on affected devices.
  • DDoS - Distributed Denial of Service, a type of attack that overwhelms a target's resources, making it unavailable to users.
Defender focus
  • Implement network monitoring to detect unusual traffic patterns indicative of DDoS activity.
  • Ensure that all TBK DVRs and TP-Link routers are updated with the latest firmware to mitigate the vulnerability.
  • Review firewall rules to block known malicious IP addresses associated with the Mirai botnet.
Do now
