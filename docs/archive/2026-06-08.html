VS Code Adds 2-Hour Extension Auto-Update Delay to Limit Supply Chain Attacks

The new two-hour delay for VS Code extension updates aims to mitigate potential supply chain attacks, making it crucial for developers to understand this change.

Who should care: Development teams using Visual Studio Code and its extensions should pay attention to this update.
Attack path: An attacker could exploit vulnerabilities in newly released extensions before they are updated, potentially injecting malicious code into the development process.
Why this matters now: A live patch or vendor fix gives defenders something concrete to validate.
In plain English: VS Code now delays automatic extension updates by two hours to enhance security against supply chain threats.
supply chain software development
  • Microsoft has implemented a two-hour delay for automatic updates of extensions in Visual Studio Code. This change is designed to provide an additional layer of protection against software supply chain attacks.
  • Supply chain attacks can compromise development environments and lead to malicious code being executed in production. By delaying updates, developers have a window to identify and mitigate any potential vulnerabilities in new extensions before they are applied.
  • An attacker could exploit vulnerabilities in newly released extensions before they are updated, potentially injecting malicious code into the development process.
  • Watch next: Monitor for any reports of vulnerabilities in popular VS Code extensions that could be exploited during the update delay.
Key terms
  • Supply Chain Attack - a type of cyber attack that targets the vulnerabilities in the supply chain of software, potentially compromising the software before it reaches the end user.
  • Extension - a software add-on that enhances the functionality of an application, in this case, Visual Studio Code.
Defender focus
  • Monitor for unusual activity related to VS Code extensions during the update window.
  • Review logs for any unauthorized changes or installations of extensions.
  • Ensure that development teams are aware of the new update delay and its implications.
Do now
  • Inform development teams about the new two-hour update delay.
  • Review and assess the security of currently installed extensions.
  • Stay updated on any vulnerabilities reported for VS Code extensions.
