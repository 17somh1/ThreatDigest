npm 12 Disables Install Scripts by Default to Reduce Supply Chain Risk

The default disabling of install scripts in npm 12 significantly reduces the risk of supply chain attacks, making it crucial for developers to adapt their workflows immediately.

Who should care: Development teams using npm for package management should prioritize this update to enhance security.
Attack path: An attacker could exploit install scripts in npm packages to execute malicious code during installation; disabling these scripts prevents such exploitation.
Why this matters now: Useful context for teams tracking where attacker pressure is building.
In plain English: npm 12 now disables install scripts by default, which helps prevent potential supply chain vulnerabilities.
supply chain npm security
  • GitHub released npm version 12, which disables install scripts by default. This change aims to mitigate risks associated with supply chain attacks. Additionally, granular access tokens (GATs) that bypass two-factor authentication (2FA) have been deprecated.
  • By disabling install scripts, npm 12 reduces the attack surface for malicious code execution during package installation. Developers must now explicitly enable scripts, which encourages safer practices and reduces the likelihood of inadvertently executing harmful code.
  • An attacker could exploit install scripts in npm packages to execute malicious code during installation; disabling these scripts prevents such exploitation.
  • Watch next: Monitor for updates from npm regarding user adoption of these new defaults and any emerging vulnerabilities.
Key terms
  • Supply Chain Attack - a type of cyber attack where an attacker targets a less secure element in the supply chain to compromise a system.
  • Install Scripts - scripts that run automatically during the installation of software packages, which can be exploited if not properly managed.
Defender focus
  • Review and update package management policies to align with npm 12 changes.
  • Monitor for any attempts to exploit outdated npm versions still using default install scripts.
  • Educate developers on the importance of reviewing and enabling install scripts only when necessary.
Do now
  • Update npm to version 12 across all development environments.
  • Audit existing npm packages for reliance on install scripts.
  • Implement a review process for enabling install scripts in new package installations.
