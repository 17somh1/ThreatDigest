Quasar Linux RAT Steals Developer Credentials for Software Supply Chain Compromise

The emergence of Quasar Linux RAT poses a significant threat to software supply chains by targeting developer credentials, which could lead to widespread compromise of software integrity.

Who should care: Development teams, DevOps engineers, and organizations relying on software supply chains should be particularly vigilant.
Attack path: Attackers gain leverage by infiltrating developer environments, allowing them to steal sensitive credentials and manipulate software development processes.
Why this matters now: Useful context for teams tracking where attacker pressure is building.
In plain English: A new Linux RAT is stealing developer credentials, risking software supply chain security.
ransomware cve cloud
  • Quasar Linux RAT (QLNX) has been identified as a previously undocumented implant targeting developers' systems. It enables attackers to harvest credentials, log keystrokes, manipulate files, monitor clipboards, and tunnel network traffic.
  • Compromised developer credentials can lead to unauthorized access to critical software components, potentially allowing attackers to insert malicious code into legitimate software. This could undermine trust in software products and disrupt operations across various industries.
  • Attackers gain leverage by infiltrating developer environments, allowing them to steal sensitive credentials and manipulate software development processes.
  • Watch next: Monitor for updates from security vendors regarding detection capabilities for QLNX and any emerging indicators of compromise.
Key terms

Defender focus

New Linux PamDOORa Backdoor Uses PAM Modules to Steal SSH Credentials

The emergence of the PamDOORa backdoor poses a significant risk to Linux systems, enabling persistent unauthorized access and credential theft.

Who should care: Linux system administrators and security teams managing SSH access should prioritize monitoring for this threat.
Attack path: Attackers can gain leverage by deploying PamDOORa to exploit PAM configurations, allowing them to authenticate as legitimate users without detection.
Why this matters now: Useful context for teams tracking where attacker pressure is building.
In plain English: A new Linux backdoor called PamDOORa allows attackers to steal SSH credentials and maintain access to compromised systems.
backdoor linux ssh
  • Researchers have identified a new backdoor named PamDOORa, which is being sold on a Russian cybercrime forum for $1,600. This backdoor utilizes Pluggable Authentication Modules (PAM) to facilitate persistent SSH access through a specific password and TCP port.
  • If exploited, PamDOORa can allow attackers to maintain long-term access to Linux systems, potentially leading to further data breaches or system compromises. Organizations using Linux servers should be particularly vigilant, as this backdoor targets SSH credentials, a common access point for remote management.
  • Attackers can gain leverage by deploying PamDOORa to exploit PAM configurations, allowing them to authenticate as legitimate users without detection.
  • Watch next: Monitor for any updates from cybersecurity researchers regarding the distribution and detection of PamDOORa.
Key terms
  • PAM - Pluggable Authentication Module, a framework that allows the integration of various authentication methods into Linux systems.
  • SSH - Secure Shell, a protocol used to securely access and manage devices over a network.
Defender focus
  • Implement monitoring for unusual SSH access patterns.
  • Review PAM configurations for unauthorized changes.
  • Ensure that SSH keys and passwords are managed securely and rotated regularly.
Do now
  • Audit PAM configurations for potential vulnerabilities.
  • Enhance logging and monitoring of SSH access attempts.
  • Educate system administrators on recognizing signs of unauthorized access.
