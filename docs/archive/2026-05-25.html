Who should care: Web development teams and organizations using Ghost CMS are most exposed to this threat.
Attack path: Attackers exploit the SQL injection vulnerability to inject malicious JavaScript, which can then execute harmful actions within the application context.
Why this matters now: Useful context for teams tracking where attacker pressure is building.
In plain English: A serious vulnerability in Ghost CMS is being actively exploited, allowing attackers to inject malicious code.

TrapDoor Supply Chain Attack Spreads Credential-Stealing Malware via npm, PyPI, and CratesIO

The TrapDoor supply chain attack is actively distributing credential-stealing malware, posing immediate risks to software developers and their projects.

Who should care: Development teams using npm, PyPI, or Crates.io should be particularly vigilant.
Attack path: Attackers gain leverage by embedding malware in legitimate-looking packages, which can be easily installed by unsuspecting developers. This allows them to steal credentials and potentially access sensitive systems.
Why this matters now: Useful context for teams tracking where attacker pressure is building.
In plain English: A coordinated attack is spreading malware through popular package managers, affecting developers using npm, PyPI, and Crates.io.
supply chain malware credential theft
  • A new supply chain attack, named TrapDoor, has been identified, targeting npm, PyPI, and Crates.io to distribute credential-stealing malware. Over 34 malicious packages have been identified, with more than 384 versions published since May 22, 2026. The attack is ongoing, with new packages being released in waves.
  • This attack can compromise developer credentials, leading to unauthorized access to repositories and potential further exploitation of affected systems. The widespread nature of the attack across multiple ecosystems increases the risk of a larger impact on software supply chains and the integrity of applications.
  • Attackers gain leverage by embedding malware in legitimate-looking packages, which can be easily installed by unsuspecting developers. This allows them to steal credentials and potentially access sensitive systems.
  • Watch next: Monitor for updates from package managers regarding the removal of malicious packages and any additional security advisories.
Key terms
  • Supply Chain Attack - a type of cyber attack that targets the supply chain of software, compromising the software before it reaches the end user.
  • Credential-Stealing Malware - malicious software designed to capture user credentials, such as usernames and passwords.
Defender focus
  • Implement monitoring for unusual package installations in development environments.
  • Review and audit dependencies for known vulnerabilities and malicious packages.
  • Educate developers on verifying package integrity and sources before installation.
Do now
  • Immediately review and audit all dependencies in your projects.
  • Implement stricter controls on package installations from npm, PyPI, and Crates.io.
  • Stay updated on security advisories from package managers and adjust security practices accordingly.
