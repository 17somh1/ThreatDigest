Public PoC Released for Critical libssh2 CVE-2026-55200 Client-Side SSH Flaw

The release of a public proof-of-concept for a critical SSH vulnerability poses an immediate risk of exploitation, potentially allowing attackers to execute arbitrary code on vulnerable client systems.

Who should care: Development and security teams using libssh2 in their applications should prioritize addressing this vulnerability.
Attack path: An attacker could set up a malicious SSH server that, when connected to by a vulnerable client, triggers the memory corruption flaw, allowing for arbitrary code execution.
Why this matters now: Useful context for teams tracking where attacker pressure is building.
In plain English: A serious flaw in the libssh2 library could let attackers run code on client machines without any user interaction.
cve ssh libssh2
  • A public proof-of-concept (PoC) has been released for CVE-2026-55200, a critical vulnerability in the libssh2 library that allows a malicious SSH server to trigger memory corruption on connecting clients. This flaw affects all versions of libssh2 up to and including 1.11.1 and has a CVSS score of 9.2, indicating severe risk.
  • This vulnerability can be exploited without user credentials or interaction, making it particularly dangerous for organizations using affected versions of libssh2. If exploited, attackers could gain control over client systems, leading to data breaches or further network compromise.
  • An attacker could set up a malicious SSH server that, when connected to by a vulnerable client, triggers the memory corruption flaw, allowing for arbitrary code execution.
  • Watch next: Monitor for updates from libssh2 regarding patches or mitigations for this vulnerability.

Microsoft Removes 119 Edge Extensions That Hid Malware in Images and Fonts

The removal of 119 malicious Edge extensions highlights a significant threat to user credentials and system integrity, necessitating immediate attention from affected teams.

Who should care: IT security teams and users of Microsoft Edge should be particularly concerned about this threat.
Attack path: Attackers leveraged steganography to conceal malware within seemingly benign files, allowing them to bypass initial security checks and activate later for credential theft.
Why this matters now: Useful context for teams tracking where attacker pressure is building.
In plain English: Microsoft has removed 119 malicious Edge extensions that were stealing user credentials and running ad fraud.
malware edge credentials
  • Microsoft has identified and removed 119 malicious extensions from the Edge Add-ons store that were designed to hide malware within image and font files. These extensions, linked to a single threat actor, had been active since at least 2021 and would activate days after installation to steal user credentials and conduct ad fraud.
  • The presence of these extensions poses a serious risk to users, as they can lead to unauthorized access to sensitive information and financial loss through ad fraud. Organizations using Edge should be vigilant as these extensions could have been installed on corporate devices, potentially compromising security.
  • Attackers leveraged steganography to conceal malware within seemingly benign files, allowing them to bypass initial security checks and activate later for credential theft.

Defender focus
