New Python Backdoor Uses Tunneling Service to Steal Browser and Cloud Credentials

The emergence of the DEEP#DOOR backdoor poses a significant threat to organizations by enabling attackers to steal sensitive credentials and maintain persistent access.

Who should care: IT security teams and organizations using Windows environments with web browsers and cloud services should be particularly concerned.
Attack path: Attackers gain leverage by disabling security controls on Windows systems, allowing them to execute the backdoor and maintain access to sensitive information.
Why this matters now: Useful context for teams tracking where attacker pressure is building.
In plain English: A new Python backdoor can disable security controls and steal credentials from browsers and cloud services.
backdoor python credential theft
  • Researchers have identified a new backdoor framework named DEEP#DOOR, which is written in Python. It begins its attack by executing a batch script that disables Windows security features and then establishes a persistent connection to the attacker’s server.
  • This backdoor can harvest sensitive information such as browser and cloud service credentials, potentially leading to data breaches and unauthorized access to critical systems. Organizations must be vigilant as the backdoor can remain undetected due to its stealthy nature.
  • Attackers gain leverage by disabling security controls on Windows systems, allowing them to execute the backdoor and maintain access to sensitive information.
  • Watch next: Monitor for any updates from security vendors regarding detection and mitigation strategies for DEEP#DOOR.
Key terms
  • Backdoor - a method of bypassing normal authentication to gain remote access to a computer.
  • Tunneling - a technique that allows data to be sent securely over a network by encapsulating it within another protocol.
Defender focus
  • Implement monitoring for unusual batch script executions on Windows systems.
  • Enhance endpoint security to detect and block unauthorized access attempts.
  • Review and tighten security configurations to prevent disabling of security controls.
Do now
  • Immediately review security logs for any signs of the 'install_obf.bat' script execution.
  • Update endpoint protection solutions to ensure they can detect the DEEP#DOOR backdoor.
  • Educate teams on the risks of disabling security features and the importance of maintaining security controls.
