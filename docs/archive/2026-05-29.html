The discovery of a malicious NuGet package targeting Sicoob's banking clients poses a significant risk to sensitive financial data and could lead to unauthorized access to accounts.

Who should care: Development teams using NuGet packages, especially those integrating with Sicoob's services.
Attack path: An attacker can leverage the malicious package to gain access to sensitive banking credentials by tricking developers into installing it as part of their software projects.
Why this matters now: Useful context for teams tracking where attacker pressure is building.
In plain English: A malicious NuGet package is stealing banking credentials from Sicoob clients.
malware cve cloud
  • Cybersecurity researchers found that versions 2.0.0 through 2.0.4 of a NuGet package named 'Sicoob.Sdk' are designed to exfiltrate client IDs and PFX certificates. This package pretends to be a legitimate software development kit for Sicoob, one of Brazil's largest cooperative financial systems.
  • The exfiltration of PFX certificates and client IDs can lead to unauthorized access to banking accounts, potentially resulting in financial loss for clients. As this package is distributed via NuGet, it may affect numerous developers and organizations using these libraries without awareness of the threat.
  • An attacker can leverage the malicious package to gain access to sensitive banking credentials by tricking developers into installing it as part of their software projects.
  • Watch next: Monitor for updates from Sicoob regarding the integrity of their SDK and any patches released for affected versions.
Key terms
  • NuGet - a package manager for .NET developers that allows them to share and consume code libraries.
  • PFX certificate - a file format used to store cryptographic information, including private keys and certificates.
Defender focus
  • Implement monitoring for the use of the Sicoob.Sdk package in development environments.

  • Educate developers on verifying the integrity of third-party packages before use.
Do now
