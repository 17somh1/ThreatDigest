CISA Adds Exploited Magento RCE Flaw CVE-2026-45247 to KEV Catalog

Active exploitation of a critical vulnerability in Magento could lead to severe data breaches and operational disruptions for affected businesses.

Who should care: Magento administrators and security teams managing e-commerce platforms using the Mirasvit Cache Warmer should prioritize this issue.
Attack path: An attacker can exploit this vulnerability by sending specially crafted requests to the vulnerable extension, allowing them to execute arbitrary code on the server.
Why this matters now: Active exploitation makes this an immediate exposure check. KEV status shortens the acceptable patch window.
In plain English: A critical RCE flaw in Magento is being actively exploited, posing significant risks to online retailers.
rce cve magento federal vulnerability
  • CISA has added CVE-2026-45247, a critical remote code execution (RCE) vulnerability affecting the Mirasvit Cache Warmer for Magento, to its KEV catalog. This vulnerability has a CVSS score of 9.8, indicating its severity. Reports confirm that it is currently being exploited in the wild.
  • The exploitation of this vulnerability could allow attackers to execute arbitrary code on affected systems, potentially leading to unauthorized access to sensitive data and disruption of services. Online retailers using this extension are particularly vulnerable, which could result in significant financial losses and damage to reputation.
  • An attacker can exploit this vulnerability by sending specially crafted requests to the vulnerable extension, allowing them to execute arbitrary code on the server.
  • Also reported by Alerts.
  • Watch next: Monitor for updates from CISA and Magento regarding patches or mitigations for this vulnerability.
Key terms
  • RCE - Remote Code Execution, a type of vulnerability that allows an attacker to run arbitrary code on a target system.
  • CVSS - Common Vulnerability Scoring System, a standard for assessing the severity of security vulnerabilities.
  • CVE - Common Vulnerabilities and Exposures, a list of publicly disclosed cybersecurity vulnerabilities.
  • Deserialization - The process of converting data from a format suitable for storage or transmission back into its original format, which can be exploited if untrusted data is processed.
Defender focus
  • Implement monitoring for unusual requests targeting the Mirasvit Cache Warmer.
  • Ensure that all Magento installations are updated to the latest version.
  • Review access logs for any signs of exploitation attempts.
Do now
  • Patch the Mirasvit Cache Warmer extension immediately.
  • Conduct a security audit of Magento installations for other vulnerabilities.
  • Educate the team on recognizing signs of exploitation.
Watch next: Monitor for updates from CISA and Magento regarding patches or mitigations for this vulnerability.

Autonomous AI Tool Finds 2-Year-Old RCE Flaw in Redis (CVE-2026-23479)

A critical remote code execution flaw in Redis could allow attackers to execute arbitrary commands, posing significant risks to database integrity and security.

Who should care: Database administrators and security teams managing Redis installations should prioritize this issue.
Attack path: An attacker with authenticated access could exploit this vulnerability to execute arbitrary commands on the server, potentially leading to full system compromise.
Why this matters now: A live patch or vendor fix gives defenders something concrete to validate.
In plain English: Redis has patched a serious flaw that could let authenticated users run harmful commands on the database server.
rce cve redis
  • Redis has addressed a use-after-free vulnerability in its blocking-client code, identified as CVE-2026-23479. This flaw, present since version 7.2.0, allowed authenticated users to execute arbitrary operating system commands on the database host. The vulnerability went unnoticed for over two years until it was discovered by an autonomous AI tool.
  • The existence of this flaw for such an extended period raises concerns about the security of systems using Redis, especially those that rely on authenticated access. If exploited, attackers could gain control over the database server, leading to data breaches or service disruptions.
  • An attacker with authenticated access could exploit this vulnerability to execute arbitrary commands on the server, potentially leading to full system compromise.
  • Watch next: Monitor for any unusual activity in Redis logs and stay updated on patching practices from Redis maintainers.
Key terms
  • RCE - Remote Code Execution, a vulnerability that allows an attacker to run commands on a remote machine.
  • Use-after-free - A type of vulnerability that occurs when a program continues to use a pointer after the memory it points to has been freed.
Defender focus
  • Review access logs for any unauthorized authenticated access to Redis instances.
  • Ensure all Redis installations are updated to the latest patched version.
  • Implement network segmentation to limit access to Redis databases.
Do now
  • Immediately update Redis to the latest version to mitigate the vulnerability.
  • Audit user access to Redis to ensure only authorized personnel have access.
  • Monitor for any signs of exploitation or unusual command execution on Redis servers.
Watch next: Monitor for any unusual activity in Redis logs and stay updated on patching practices from Redis maintainers.

ransomware sanctions crypto
