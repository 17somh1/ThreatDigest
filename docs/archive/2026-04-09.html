Adobe Reader Zero-Day Exploited via Malicious PDFs Since December 2025

The exploitation of a zero-day vulnerability in Adobe Reader poses a significant risk to organizations relying on this software for document handling, potentially leading to data breaches.

Who should care: IT security teams and organizations using Adobe Reader for document management should prioritize this issue.
Attack path: Attackers can deliver the exploit through crafted PDF files, which, when opened, can execute malicious code on the victim's machine.
Why this matters now: Multiple sources point to the same operational problem.
In plain English: A serious vulnerability in Adobe Reader is being actively exploited through malicious PDFs, affecting users since late 2025.
cve pdf adobe exploit
  • Threat actors have been leveraging a zero-day vulnerability in Adobe Reader to exploit users via malicious PDF documents since at least December 2025. The exploit was first identified through a file named 'Invoice540.pdf' that appeared on VirusTotal in late November 2025.
  • This vulnerability allows attackers to execute arbitrary code on affected systems, which can lead to unauthorized access and data exfiltration. Organizations using Adobe Reader are at heightened risk, especially if they handle sensitive information in PDF format.
  • Attackers can deliver the exploit through crafted PDF files, which, when opened, can execute malicious code on the victim's machine.
  • Also reported by BleepingComputer.
  • Watch next: Monitor for updates from Adobe regarding patches or mitigations for this vulnerability.
Key terms
  • ZERO-DAY - a vulnerability that is exploited before the vendor has released a fix.
  • EXPLOIT - a method used by attackers to take advantage of a vulnerability.
  • Zero-day vulnerability - a security flaw that is exploited before the vendor has released a fix.
  • Malicious PDF - a PDF file that contains harmful code designed to compromise a user's system.
Defender focus
  • Implement monitoring for unusual PDF file activity within your network.
  • Ensure that all instances of Adobe Reader are updated to the latest version as soon as a patch is released.
  • Educate users on the risks of opening unsolicited PDF attachments.
Do now
