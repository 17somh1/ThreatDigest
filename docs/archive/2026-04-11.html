Google Chrome adds infostealer protection against session cookie theft

The introduction of Device Bound Session Credentials in Chrome is crucial as it directly mitigates the risk of session cookie theft, a common attack vector for infostealers.

Who should care: Security teams and IT departments managing Chrome deployments should prioritize this update.
Attack path: Attackers typically deploy infostealer malware to harvest session cookies from compromised devices. With DBSC, even if malware is present, it cannot access these cookies.
Why this matters now: Multiple sources point to the same operational problem.
In plain English: Google Chrome has implemented new protections to prevent session cookie theft by malware.
infostealer cve browser security chrome session theft
  • Google has released Chrome version 146 for Windows, which includes a new feature called Device Bound Session Credentials (DBSC). This feature is designed to prevent info-stealing malware from accessing session cookies, which are often targeted in attacks.
  • Session cookies are critical for maintaining user sessions and can be exploited to hijack accounts. By blocking access to these cookies, the new protection reduces the risk of unauthorized access to sensitive accounts and data, which is particularly important for organizations handling confidential information.
  • Attackers typically deploy infostealer malware to harvest session cookies from compromised devices. With DBSC, even if malware is present, it cannot access these cookies.
  • Also reported by The Hacker News.

  • Session Cookies - small pieces of data stored on a user's device that keep them logged into websites.
  • Device Bound Session Credentials - a security feature that ties session credentials to a specific device, reducing the risk of session hijacking.
  • Session Hijacking - a type of attack where an attacker takes control of a user's session to gain unauthorized access.
