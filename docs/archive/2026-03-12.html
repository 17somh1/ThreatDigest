Apple Issues Security Updates for Older iOS Devices Targeted by Coruna WebKit Exploit

Older iOS devices are now at risk from a serious WebKit exploit, making immediate updates essential to prevent potential breaches.

Who should care: IT and security teams managing older Apple devices should prioritize these updates to mitigate risks.
Attack path: An attacker could leverage the WebKit vulnerability to execute malicious code on an unpatched device, gaining unauthorized access to sensitive information.
Why this matters now: A live patch or vendor fix gives defenders something concrete to validate.
In plain English: Apple has released security updates for older iOS devices to address a critical WebKit vulnerability exploited by attackers.
cve ios security
  • Apple issued security updates for older versions of iOS, iPadOS, and macOS Sonoma to address a vulnerability tracked as CVE-2023-43010. This flaw, related to WebKit, can lead to memory corruption when handling malicious web content. The updates were backported after the vulnerability was identified as part of the Coruna exploit kit.
  • If left unpatched, this vulnerability could allow attackers to execute arbitrary code on affected devices, potentially leading to data theft or device compromise. Given the widespread use of older iOS devices, the risk of exploitation is significant, especially in environments where these devices are still operational.
  • An attacker could leverage the WebKit vulnerability to execute malicious code on an unpatched device, gaining unauthorized access to sensitive information.
  • Watch next: Monitor for reports of exploitation attempts using the Coruna exploit kit and ensure all devices are updated promptly.
Key terms
  • CVE-2023-43010 - a unique identifier for a specific security vulnerability in software.
  • WebKit - a browser engine used by Apple for rendering web pages.
Defender focus
  • Verify that all older iOS devices are updated to the latest version.
  • Implement monitoring for unusual behavior on devices that may indicate exploitation attempts.
  • Educate users about the importance of applying updates promptly.
Do now
  • Immediately apply the latest security updates to all affected devices.
  • Conduct a risk assessment to identify any unpatched devices in your environment.
  • Review security policies to ensure timely updates are part of routine maintenance.
